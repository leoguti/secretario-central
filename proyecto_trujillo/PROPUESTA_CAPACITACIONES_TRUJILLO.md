## Capacitación 2: Administración técnica de la plataforma

Cómo operar y mantener los servidores, la app móvil y el planificador de rutas.

### La plataforma usa estas tecnologías

- **App móvil:** Flutter (lenguaje Dart)
- **Planificador de rutas:** OpenTripPlanner (Java)
- **Servidores:** Ubuntu Linux
- **Contenedores:** Docker
- **Servidor web:** Nginx

### Requisitos previos del personal técnico

> **Esto es fundamental.** La persona que administre esta plataforma necesita tener conocimientos previos en las áreas listadas abajo. Sin estas bases, no es posible hacer una transferencia tecnológica efectiva. No cualquier persona puede administrar un sistema como este.

**1. Linux (obligatorio)**
- Saber usar la terminal (línea de comandos)
- Manejar archivos y permisos
- Instalar y actualizar software
- Administrar servicios del sistema
- Leer logs para diagnosticar problemas

**2. Redes (obligatorio)**
- Entender puertos, HTTP/HTTPS
- Configurar firewall
- Manejar dominios y certificados SSL
- Conectarse por SSH

**3. Docker (obligatorio)**
- Entender qué son contenedores e imágenes
- Levantar, detener y actualizar servicios con Docker
- Usar Docker Compose

**4. Servidor web - Nginx (obligatorio)**
- Configurar Nginx como proxy reverso
- Manejar HTTPS
- Leer logs del servidor

**5. Git (obligatorio)**
- Clonar repositorios, hacer pull y push
- Manejar ramas

**6. Flutter (deseable)**
- Entender cómo se compila la app para Android y web
- Manejar dependencias
- Publicar en Play Store / App Store

**7. OpenTripPlanner (se enseña en la capacitación)**
- Cargar datos GTFS y mapas
- Configurar y reiniciar el servicio
- Monitorear estado del servidor
